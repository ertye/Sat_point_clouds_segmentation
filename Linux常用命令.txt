当协作的同事（其他人）在远程仓库（例如 GitHub）上修改并推送了代码后，您需要执行 **拉取（Pull）** 操作来同步这些更改，将最新的代码更新到您的本地分支。

有两种主要的更新方式：**合并 (Merge)** 和 **变基 (Rebase)**。

-----

## 方式一：合并 (Merge) —— 保留历史记录

这是最直接、最常用的方法。它将远程的更改和您的本地更改合并，并生成一个特殊的 **合并提交 (Merge Commit)**。

### 命令：`git pull`

```bash
# 确保您在要更新的本地分支上 (例如 main)
git checkout main

# 执行拉取操作 (等同于 git fetch + git merge)
git pull origin main
```

**效果：**

1.  Git 会从远程仓库 `origin` 获取 `main` 分支的最新数据。
2.  Git 会尝试将远程的最新提交与您本地的提交进行合并。
3.  如果合并成功，将生成一个新的 **Merge Commit**。您的 Git 历史记录会显示两条分支线汇合。

-----

## 方式二：变基 (Rebase) —— 保持历史记录线性

变基是一种“清理”历史记录的方法。它将您的本地提交暂时“拿走”，更新远程的最新提交，然后将您的本地提交重新应用（放在）在远程提交的**后面**。这样可以保持 Git 历史记录的整洁和线性。

### 命令：`git pull --rebase`

```bash
# 确保您在要更新的本地分支上
git checkout main

# 执行拉取并变基操作
git pull origin main --rebase
```

**效果：**

1.  Git 获取远程最新提交。
2.  Git 将您的本地提交临时保存。
3.  将远程提交作为新的起点。
4.  将您的本地提交一个个重新应用到新的起点上。
5.  **不会**产生 Merge Commit，历史记录看起来就像是您一直基于最新代码进行开发。

-----

## 发生冲突 (Conflict) 时的处理

无论您使用 `git pull` 还是 `git pull --rebase`，如果您的本地修改和远程的修改在同一个文件的同一部分，就会发生冲突。

### 冲突解决步骤：

1.  **Git 暂停：** 命令行会提示您发生冲突，并列出冲突的文件。
2.  **手动解决冲突：**
      * 使用代码编辑器打开冲突文件。
      * 查找文件中类似 `<<<<<<<`, `=======`, `>>>>>>>` 的标记。
      * 手动修改文件内容，选择您想保留的代码（通常是合并您和同事的代码），并删除这些标记。
3.  **标记为已解决：**
    ```bash
    git add .
    ```
4.  **继续操作：**
      * **如果您使用的是 `git pull` (Merge)：**
        ```bash
        git commit -m "Merge remote-tracking branch 'origin/main'" 
        # 默认的提交信息即可
        ```
      * **如果您使用的是 `git pull --rebase` (Rebase)：**
        ```bash
        git rebase --continue
        ```

完成这些步骤后，您的本地分支就会是最新的代码。
